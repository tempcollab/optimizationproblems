# Bóna — the coloring / word-encoding upper bounds

- B14a "A new upper bound for 1324-avoiding permutations." arXiv:1207.2379; CPC 2014.
  L(1324) ≤ 7+4√3 ≈ 13.928. PDF: `pdfs/bona2014.pdf`.
- B14b "A new record for 1324-avoiding permutations." arXiv:1404.4033; EJM 2015.
  L(1324) ≤ 13.73977. PDF: `pdfs/bona2015.pdf`.

## The method (shared, and the most Lean-fit upper-bound route)

**Coloring.** Given a 1324-avoider p, colour each entry red or blue by a greedy rule:
- (B14a) color p_i blue if coloring it red would create an all-red 132, or if there is already
  a smaller blue entry; else red.  (The red entries avoid 132; the structure is captured.)
**Encoding.** The colouring + relative-order data is encoded as a **pair of words (w(p), z(p))**
of length n over a 4-letter alphabet {A,B,C,D} (the letters mark colour/role of each entry).
The map p ↦ (w(p), z(p)) is an **injection** (B14b Thm 2.3 / 3.x).

**Constraints (the source of the improvement).** Not every word pair arises from a
1324-avoider; the image satisfies local forbidden-factor constraints:
- **No CB-factor** (B14a Lemma 3.2): captures one consequence of 1324-avoidance.
- **CAB^k-factor ⟹ ≥ k B's in the corresponding z-segment** (B14b Lemma 3.1, 4.1):
  a refined family of constraints, one per k.

**Bound = a rational generating function radius.** Count word-pairs satisfying the
constraints; the count h_n (or k_n, t_n) has an **explicit rational generating function**
whose denominator's smallest-modulus root α gives β = 1/α, and L(1324) ≤ β^2:
- Using CB-factor only: H(x) rational (B14b Thm 3.3), root α=0.2695867676, β=3.709381,
  L ≤ β^2 = 13.7595.
- Adding CAB^2: K(x), β=3.70672, **L ≤ 13.73977** (B14b Thm 4.4).
- Using all CAB^k (restriction (vi)): T(x) rational (B14b Thm 4.5) — but author notes the
  improvement over 13.74 is "minuscule"; the method **saturates around 13.7**.

**Why it saturates / where it is loose.** The (w,z) encoding throws away global 1324-structure
and keeps only *local* forbidden factors; many word-pairs satisfying all CAB^k constraints
still do not come from genuine 1324-avoiders. Capturing more constraints simultaneously makes
the word-counting intractable. The BBEPP staircase/domino method beat this (13.5) by using
*global* structure instead.

## Lean-fit assessment — HIGH
This is the **most Lean-fit of the existing upper-bound methods**: the load-bearing object is
an **explicit rational generating function** and the bound is "the smallest-modulus root of an
explicit integer polynomial is ≥ some rational r ⟹ growth ≤ 1/r^... ". Verifying a root bound
on a fixed polynomial is a finite algebraic certificate (Sturm sequences / interval arithmetic
on rationals — directly formalisable). The *combinatorial* part (injection + which factors are
forbidden) is a finite case argument on words. A NEW forbidden-factor / encoding refinement
that drops β^2 below 13.5 would be a record AND Lean-certifiable. Caveat: matching/beating
BBEPP's 13.5 by this route requires capturing *more* structure than CAB^k — that is exactly
where Bóna stalled, so the hard step is finding the additional certifiable constraint, not the
GF/root machinery.
